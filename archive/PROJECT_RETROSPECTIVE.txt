ATTENTION DROP DETECTOR - PROJECT RETROSPECTIVE (V6)
====================================================

This document outlines the development journey of the Attention Drop Detector from its initial V5 state to the robust V6, detailing the challenges encountered, the debugging process, and the solutions implemented. This project serves as a strong example of practical AI integration, system-level debugging, and robust application packaging.


PROJECT OVERVIEW
----------------
The Attention Drop Detector is an automated video analysis tool designed to identify moments of potential viewer disengagement in video content. It analyzes videos using a combination of:
-   **OpenCV:** For motion tracking (Farneback Optical Flow).
-   **Librosa:** For audio energy and speech probability analysis.
-   **MediaPipe:** For advanced face detection (FaceMesh).
-   **PySceneDetect:** For identifying scene cuts.
The tool provides a web-based UI (Flask) and a command-line interface, offering insights into video pacing, engagement, and areas for improvement.


INITIAL STATE (V5) - PROBLEMS ENCOUNTERED
-----------------------------------------
Upon initial review of the V5 codebase, several critical issues were identified:

1.  **Audio Energy Always 0.00:**
    *   **Symptom:** The audio analysis consistently reported 0.00 for energy, indicating a failure in processing audio from MP4 files.
    *   **Diagnosis:** `librosa.load()` on Windows, when dealing with MP4 containers, relies on `audioread` which in turn depends on `ffmpeg` being correctly installed and accessible in the system's PATH. A misconfigured or missing `ffmpeg` installation would cause silent failures or empty audio arrays.

2.  **Face Detection Returning 0%:**
    *   **Symptom:** Despite faces being present in the video, MediaPipe's face detection reported 0% presence.
    *   **Diagnosis (Initial):** The `FaceMesh` model was initialized with `static_image_mode=False`. While this is suitable for continuous video streams, the `analyzer.py` script was performing `cap.set()` to seek to specific timestamps and skipping frames. This broke the temporal continuity MediaPipe expected for tracking, causing it to lose faces immediately.
    *   **Diagnosis (Deeper - Python 3.13 Compatibility):** Further investigation revealed a more fundamental issue. The user's system was running Python 3.13 (a very new release), for which Google's MediaPipe had not yet released stable pre-compiled C++ binaries. This led to `mediapipe` silently failing to load its core components, resulting in `module 'mediapipe' has no attribute 'solutions'` errors, and forcing a fallback to the less robust Haar Cascade detector.

3.  **EXE Build Using PyInstaller Had Missing Dependencies:**
    *   **Symptom:** Previous attempts to build a standalone executable (`.exe`) using PyInstaller resulted in missing module warnings and runtime failures.
    *   **Diagnosis:** Complex libraries like MediaPipe, PySceneDetect, and Librosa often load components dynamically or have hidden dependencies that PyInstaller's default analysis misses.


SOLUTIONS IMPLEMENTED (V6)
--------------------------

1.  **Robust Audio Extraction (FFmpeg Subprocess):**
    *   **Fix:** Modified `analyzer.py` to explicitly use `subprocess` to call `ffmpeg`. It now extracts a temporary uncompressed `.wav` file from the input video, which `librosa` can reliably load. This bypasses `audioread`'s potential issues with `ffmpeg` PATH configuration.
    *   **Enhancement:** Added `creationflags=subprocess.CREATE_NO_WINDOW` for Windows builds to prevent a black console window from flashing when `ffmpeg` is called by the `.exe`.
    *   **User Guidance:** Included `winget install ffmpeg` instructions in `README.md` and `BUILD.md` for easy installation.

2.  **MediaPipe FaceMesh Reliability & Python Version Management:**
    *   **Fix 1 (Temporal Continuity):** Changed `_mp_face_mesh.FaceMesh` initialization to `static_image_mode=True`. This forces MediaPipe to perform a fresh, robust detection on each sampled frame, regardless of seeking or skipped frames.
    *   **Fix 2 (Python 3.13 Incompatibility):**
        *   **Debugging:** Implemented detailed `try-except` blocks with `traceback.print_exc()` to expose the silent MediaPipe loading failures, confirming the Python 3.13 incompatibility.
        *   **Solution:** Guided the user to install Python 3.12 and set up a dedicated **Virtual Environment (`venv`)**. This isolated the project to a compatible Python version, preventing conflicts with system-wide installations.
        *   **Dependency Pinning:** Pinned `mediapipe==0.10.14` in `requirements.txt` to ensure a known stable version is always installed, avoiding issues with newer, potentially unstable releases.
    *   **Warning Suppression:** Added `os.environ['TF_CPP_MIN_LOG_LEVEL'] = '3'` to suppress verbose TensorFlow/MediaPipe C++ warnings for a cleaner console output.

3.  **PyInstaller Executable Packaging:**
    *   **Fix:** Utilized PyInstaller's `--collect-all` flag for `mediapipe`, `scenedetect`, and `librosa` to explicitly instruct it to gather all necessary hidden dependencies and data files.
    *   **Build Strategy:** Employed `--onedir` for the build. While `--onefile` creates a single executable, `--onedir` (which creates a folder containing the `.exe` and its dependencies) offers faster startup times and greater reliability for applications with large, complex libraries like MediaPipe and OpenCV.
    *   **Data Files:** Correctly included `templates` and `static` folders using `--add-data` flags.

4.  **Project Documentation & Version Control:**
    *   **`README.md`:** Significantly updated to include detailed feature descriptions, a "Technical Challenges Overcome" section (tailored for portfolio presentation), clear installation instructions (including `ffmpeg` and `venv` setup), and usage guides.
    *   **`BUILD.md`:** Created/updated with precise, step-by-step instructions for building the `.exe` within the recommended virtual environment, including all PyInstaller flags.
    *   **`requirements.txt`:** Created to lock down exact dependency versions, ensuring reproducibility across different environments.
    *   **`.gitignore`:** Implemented to prevent unnecessary files (like `venv/`, `build/`, `dist/`, `*.spec`) from being committed to the Git repository, keeping the codebase clean and focused.
    *   **Git Workflow:** Guided through `git init`, `git add`, `git commit`, `git config` (for user identity), `git branch`, `git remote add`, and `git push` (including `--force` to resolve remote conflicts), culminating in a new GitHub release.


KEY LEARNINGS & TECHNICAL CHALLENGES OVERCOME (PORTFOLIO NOTES)
---------------------------------------------------------------
This project presented several real-world engineering challenges that were systematically addressed:

1.  **Audio Decoding & FFmpeg Resiliency:** Standard audio loaders (`librosa`/`audioread`) silently failed on Windows MP4s due to missing system bindings. I built a robust `subprocess` pipeline that safely interfaces with FFmpeg to extract temporary uncompressed `.wav` files, guaranteeing accurate spectral energy scoring.
2.  **MediaPipe C++ Binding Crashes:** Discovered that Google's MediaPipe lacks pre-compiled C++ binaries for the newly released Python 3.13, causing silent module failures. I engineered custom traceback error handling to detect this environment mismatch and strictly isolated the app in a locked Python 3.12 Virtual Environment (`venv`).
3.  **Temporal Tracking Limitations:** Seeking to specific video timestamps (`cap.set()`) broke the temporal continuity expected by MediaPipe's tracking algorithms (resulting in 0% face detection). Fixed by explicitly forcing `static_image_mode=True` to treat every sampled frame as a fresh inference.
4.  **PyInstaller Executable Packaging:** Freezing heavy AI libraries into a standalone `.exe` caused missing dynamic DLLs and hidden imports. Solved by mapping comprehensive `--collect-all` hooks for `mediapipe`, `scenedetect`, and `librosa` directly into the build pipeline, and ensuring proper data file inclusion.
5.  **Cross-Platform Compatibility (Implicit):** Addressed Windows-specific issues (FFmpeg PATH, PowerShell commands, line endings) to ensure a smooth development and deployment experience.


RUNNING THE APPLICATION
-----------------------

**1. Raw Python (Development/Testing):**
   - Ensure `ffmpeg` is installed and in PATH.
   - Activate the `venv` (`.\venv\Scripts\activate`).
   - Run `python app.py` (for Web UI) or `python analyzer.py your_video.mp4` (for CLI).

**2. Standalone Executable (`.exe`):**
   - Locate `dist/app/app.exe`.
   - Double-click `app.exe`.
   - (Note: `ffmpeg` still needs to be installed on the user's system for audio analysis to function in the `.exe`).
